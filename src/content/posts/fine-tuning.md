## 什么是微调？

大模型微调是利用特定领域的数据集对已预训练的大模型进行进一步训练的过程。它旨在优化模型在特定任务上的性能，使模型能够更好地适应和完成特定领域的任务。

## 为什么需要微调？

**1. 从头训练成本极高**

大模型参数量动辄百亿千亿，训练成本可达数百万甚至上亿美元。每家公司从头训练自己的大模型性价比极低。微调让我们站在巨人的肩膀上，用极低成本获得专用模型。

**2. Prompt Engineering 有明显局限**

Prompt 是最容易上手的大模型使用方式，但缺点显著：

- 大模型对输入序列长度有限制，复杂 Prompt 容易被截断，导致输出质量下降
- **推理成本与 Prompt 长度的平方正相关** —— Prompt 越长，推理越贵
- 对个人用户影响不大，但企业级服务中推理成本是不可忽视的因素

微调将原本需要塞进 Prompt 的上下文"内化"到模型参数中，大幅降低推理成本。

**3. 自有数据能显著提升领域能力**

当 Prompt 效果达不到要求，而企业又积累了高质量的自有数据时，微调可以让模型在特定领域产生质的飞跃。

**4. 个性化服务需求**

针对每个用户的数据训练轻量级微调模型，可以实现千人千面的 AI 体验。

**5. 数据安全合规**

当数据不能传递给第三方大模型服务时，必须搭建自有模型。开源模型通常需要用自有数据微调才能满足业务需求，数据全程不出本地。

## 微调 vs Prompt vs RAG

| 维度 | Prompt | RAG | 微调 |
|------|--------|-----|------|
| 核心思路 | 优化输入方式 | 外挂知识库检索 | 修改模型参数 |
| 是否改模型 | 不改 | 不改 | 改 |
| 额外数据 | 不需要 | 需要知识库 | 需要标注数据 |
| 实时性 | 动态 | 动态（知识库可随时更新） | 静态（训练后固定） |
| 灵活性 | 高 | 高 | 低（需重新训练） |
| 推理成本 | 高（长 Prompt 贵） | 中 | 低 |
| 训练成本 | 无 | 低（建知识库） | 高（需要 GPU） |
| 适用场景 | 通用对话、日常任务 | 需要实时/私有数据的场景 | 专业领域、企业服务、风格定制 |

## 微调方法：SFT vs DPO

大语言模型的训练流程通常分三步：

**1. 预训练（Pre-Training）**

模型在大规模语料上学习语法、逻辑和常识知识。我们使用的开源模型（Qwen、Llama、DeepSeek）都已完成这一步。

**2. 监督微调（Supervised Fine-Tuning, SFT）**

用"输入-输出"格式的标注数据训练模型，让它在特定领域的输出形式和内容上表现更优。比如用中医问答数据对模型进行微调，使其能专业地回答中医相关问题。

**3. 偏好优化（Preference Optimization）**

SFT 后的模型可能出现语法正确但不符合事实或人类价值观的回复。偏好优化用于与人类价值观对齐。主流方法有两种：

- **PPO**（基于强化学习）：需要训练奖励模型，训练不够稳定
- **DPO**（直接偏好优化）：无需奖励模型，训练数据为三元组（prompt, chosen, rejected），直接让模型在"好回答"上的概率高于"差回答"。DPO 训练更稳定，是当前主流选择

## 微调技术：全参 vs LoRA vs QLoRA

| 技术 | 原理 | 适用场景 |
|------|------|----------|
| **全参微调** | 更新模型所有参数 | 复杂任务，计算资源充足 |
| **LoRA** | 冻结原始参数，在旁路新增低秩矩阵进行训练 | 简单任务、小数据量、资源有限 |
| **QLoRA** | 在 LoRA 基础上，将模型量化为 4bit/8bit 存储 | 显存极度紧张的场景 |

**LoRA 的核心思想**：在模型的参数矩阵（m x n 维）旁新增一条由两个低秩矩阵（m x r 和 r x n，r 远小于 m 和 n）构成的旁路。训练时只更新旁路参数，原始参数保持冻结。由于旁路参数量远小于原始矩阵，训练开销大幅降低。

**QLoRA** 进一步在加载时将模型量化为 4bit/8bit，计算时再反量化为 16bit，进一步降低显存消耗。代价是量化和反量化的过程会增加训练时间。

**如何选择：**
- 复杂任务 → 全参微调
- 简单任务 / 小数据量（几百到几千条）→ LoRA 或 QLoRA
- 显存极度紧张 → QLoRA

## 微调的完整流程

```
选定预训练模型 → 准备标注数据集 → 微调前测试 → 设定超参数 → 执行训练 → 微调后测试 → 效果对比
                                                                    ↓（不满意）
                                                          调整数据集/超参数，重新训练
```

## 关键超参数

**学习率（Learning Rate）**

每次迭代中参数更新的幅度。过大可能无法收敛，过小训练时间过长或陷入局部最优。通常配合 AdamW 优化器使用。

**训练轮数（Epochs）**

完整遍历训练数据集的次数。过少会欠拟合，过多会过拟合。推荐 2~10 个 epoch。数据量少可增加轮数，数据量大通常 2 个 epoch 即可。可通过观察验证集准确率变化来决定何时停止。

**批量大小（Batch Size）**

每次迭代使用的训练数据数量。主要用于调节训练速度，不应用于调节训练效果。理想值是硬件能支持的最大值。如果显存不够，可用梯度累积（gradient accumulation）来模拟更大的有效 batch size。

**序列长度（Seq Length）**

模型接受的单个训练样本的 token 序列长度。超过会被截断，不足会被填充。需根据训练数据的实际 token 长度分布来设置。

**LoRA 秩（lora_rank）**

控制 LoRA 旁路矩阵的维度。默认值通常为 32，低秩适配矩阵往往已经足够，增大秩不一定带来更好的效果。

**LoRA 缩放系数（lora_alpha）**

控制 LoRA 矩阵的影响力。数据量少时适当增大，数据量多时适当减小。取值一般为 lora_rank 的 1/2 到 2 倍。

## 一句话总结

**微调不是让模型更聪明，是让模型在某个方向上更专业——把"万事通"变成"专家"。**
